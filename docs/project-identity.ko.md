# 프로젝트 정체성 재정의

## 한 줄 정의

이 프로젝트는 여러 채용 사이트의 IT·개발 채용 공고를 사용자가 수동으로 실행하여 수집하고, raw 데이터와 정규화 데이터를 함께 저장한 뒤 웹에서 통합 조회할 수 있게 하는 수집 시스템이다.

## 프로젝트가 하는 일

- 사람인, 잡코리아, LinkedIn, 원티드, 점핏, 리멤버 등 여러 사이트의 IT·개발 공고를 수집한다.
- 가능하면 공고 목록뿐 아니라 상세 페이지까지 읽어 상세 채용 정보를 저장한다.
- raw HTML, XML, JSON 같은 원문 데이터와 정규화된 공고 데이터를 함께 저장한다.
- AI를 사용해 직군, 기술 스택, 요구사항, 주요 업무, 복지 등을 정리한다.
- 웹 콘솔에서 수집 실행, 실행 이력, 공고 상세, raw snapshot 조회를 제공한다.
- 추후 다른 서비스 연동이나 통계 분석에 사용할 수 있는 데이터 기반을 만든다.

## 프로젝트가 하지 않는 일

- 무제한 웹 탐색을 수행하는 범용 AI 브라우저를 목표로 하지 않는다.
- 계정 로그인 자동화나 지원서 제출 자동화를 기본 기능으로 두지 않는다.
- 접근 제한을 우회하는 비공식 수집기를 핵심 가치로 삼지 않는다.
- AI가 수집 파이프라인 전체를 대체하는 구조를 목표로 하지 않는다.

## 기본 사용자 흐름

1. 사용자가 웹 화면에서 수집 설정을 저장한다.
2. 사용자가 수동으로 수집 실행을 요청한다.
3. 수집기는 사이트별 adapter를 통해 공고 목록과 상세 페이지를 수집한다.
4. raw 데이터는 실행 단위로 저장하고, 정규화 데이터는 DB에 upsert 한다.
5. AI 정리 결과를 함께 저장한다.
6. 사용자는 웹 화면에서 실행 이력, 공고 상세, raw snapshot을 조회한다.

## 시스템 구성

- `Collector Core`
  사이트별 adapter, sitemap, API, 정적 HTML, 브라우저 기반 수집을 담당한다.
- `Storage Layer`
  raw blob 저장소와 정규화 DB 저장을 담당한다.
- `Normalization Layer`
  제목, 회사, 지역, 경력, 상세 본문, 기술 스택 등 공통 필드를 만든다.
- `AI Enrichment Layer`
  IT 공고 여부 판정, 직군 분류, 요약, 요구사항 추출을 수행한다.
- `Web Console`
  수동 실행, 실행 이력, 공고 상세, raw 조회를 제공한다.

## AI의 역할

AI는 수집기의 대체물이 아니라 보조 계층이다.

- 상세 본문에서 구조화된 정보 추출
- IT·개발 공고 여부 판단
- 직군 및 기술 스택 분류
- 운영자용 품질 점검과 실패 원인 요약

즉 이 프로젝트의 본체는 `데이터 수집 시스템`이고, AI는 `정리와 보조 분석 계층`이다.

## 데이터 원칙

- raw 데이터는 재처리 가능해야 한다.
- 정규화 데이터는 조회와 분석에 최적화되어야 한다.
- 중복 기준은 URL 중심으로 관리하되 사이트별 식별 정보도 보존한다.
- 실행 이력과 최신 상태를 모두 추적할 수 있어야 한다.

## 수집 전략 우선순위

`공개 API > sitemap > 정적 HTML > 브라우저 렌더링 > 외부 검색 fallback`

접근 제한이 강한 사이트는 best-effort 로 구분하고, 지원 범위를 명시적으로 관리한다.

## 성공 기준

- 사용자가 하루에 한 번 또는 필요할 때 수동 실행하여 여러 사이트의 IT 공고를 안정적으로 수집할 수 있다.
- raw 데이터와 정규화 데이터가 함께 보존된다.
- 웹에서 사이트 구분 없이 통합 조회가 가능하다.
- 공고 상세 정보를 분석에 바로 활용할 수 있다.
