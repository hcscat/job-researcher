# 프로필 기반 채용공고 수집/조회 계획

## 목적

- `portfolio_profile_base_20260419.md` 기준으로 사용자의 실제 경력과 맞는 공고를 우선 수집한다.
- 수집 이후에는 단순 목록이 아니라 `적합도`, `핵심 매칭 근거`, `상세 내용` 중심으로 빠르게 검토할 수 있게 만든다.
- 수집은 AI 에이전트가 수동으로 실행하되, 서버는 프로필 프리셋과 조회 화면을 제공하는 구조로 유지한다.

## 프로필 해석

- 우선 직무:
  - Java/Spring 기반 백엔드
  - 업무 시스템/운영개발
  - 공공, 이커머스, 백오피스, 관리자 화면
  - SAP UI5/Fiori, Nexacro, exBuilder 계열
- 강한 스택:
  - Java, Spring, MyBatis, JavaScript, JSP
  - Oracle, MS-SQL
  - Nexacro, exBuilder, SAP UI5/Fiori
- 보조 스택:
  - PostgreSQL, Linux, Shell Script, Selenium, Jenkins, Git
- 피해야 할 방향:
  - 데이터 사이언스, AI 리서치, 모바일 앱 전담
  - DevOps/SRE 전담
  - 디자이너/퍼블리셔/마케팅/영업

## 수집 전략

1. 프로필 프리셋 적용
- 검색 사이트는 국내 주요 채용 사이트 전체를 유지한다.
- broad scan 대신 `query_search` 중심으로 전환한다.
- 프로필 강점이 드러나는 검색어 묶음을 사용한다.

2. 검색어 구성
- `Java Spring 업무 시스템 개발자`
- `Java Spring 운영개발`
- `Java MyBatis 공공 시스템 개발`
- `Java JSP Nexacro 개발자`
- `exBuilder Java Oracle 개발`
- `SAP UI5 Fiori JavaScript 개발`
- `이커머스 운영개발 Java`
- `백오피스 Java 개발`

3. 제외어 적용
- 디자이너, 퍼블리셔, 마케팅, 영업, 간호, 교수

4. 수집 후 정리
- URL 기준으로 기존 공고와 중복을 제거한다.
- 상세 페이지가 제한된 경우에도 목록 정보와 AI 요약을 조합해 읽을 수 있는 설명을 만든다.
- 적합도 점수와 매칭 근거를 함께 저장해 조회 단계에서 다시 정렬할 수 있게 한다.

## 조회 개선 전략

- 기본 정렬을 `프로필 적합도` 기준으로 둔다.
- `추천 공고만 보기` 토글을 제공한다.
- 목록에서 제목을 클릭하면 같은 행 아래에 상세 레이어가 열리도록 유지한다.
- 상세 레이어 상단에 아래 정보를 우선 노출한다.
  - 적합도 점수
  - 적합도 레벨
  - 매칭 사유
  - 핵심 하이라이트

## 이번 작업 범위

1. 프로필 원본 파일을 저장소 루트에 복사하되 Git 추적에서는 제외한다.
2. 프로필 기반 프리셋 생성 코드와 적합도 계산 로직을 서버에 연결한다.
3. `/jobs`, `/api/jobs`, `/api/jobs/{id}` 에 적합도 필드와 추천 필터/정렬을 연결한다.
4. 채용공고 목록 UI에 프로필 요약 패널, 추천 토글, 적합도 열, 상세 레이어 적합도 섹션을 반영한다.
5. 테스트를 보강한 뒤 프로필 프리셋으로 실제 수집을 실행한다.
